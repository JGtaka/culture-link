# 管理画面 ユーザー管理 手動テストパターン

## テスト準備

- 管理者アカウント（role=admin）と一般ユーザーアカウント（role=general）を用意
- `docker compose up` でサーバー起動済み
- 事前に **一覧の件数確認用** に25件以上のユーザーを登録しておく（ページネーション確認用）
- 事前に **タブ表示確認用** に以下を1〜複数件作成しておく
  - 対象ユーザーの `QuizResult`（completed / in_progress 両方）
  - 対象ユーザーの `Favorite`（Character / Event 両方）

---

## 1. アクセス認可

### 1-1. 未ログイン
- [ ] `/admin/users` にアクセス → ログインページへリダイレクトされること
- [ ] `/admin/users/:id` にアクセス → ログインページへリダイレクトされること
- [ ] 直接 `PATCH /admin/users/:id/suspend` を叩いても停止されずログインページへリダイレクトされること

### 1-2. 一般ユーザー
- [ ] `/admin/users` にアクセス → ルート（/）へリダイレクトされること
- [ ] 「管理者権限が必要です」のフラッシュが表示されること
- [ ] 直接 `PATCH /admin/users/:id/suspend` を叩いても停止されずリダイレクトされること

### 1-3. 管理者
- [ ] サイドバーの「ユーザー管理」リンクが `/admin/users` にリンクされていること
- [ ] `/admin/users` にアクセス → 一覧ページが表示されること

## 2. ユーザー一覧ページ（`/admin/users`）

### 2-1. ページ表示
- [ ] パンくずリスト「ホーム > ユーザー管理」が表示されること
- [ ] ページタイトル「ユーザー一覧」が表示されること

### 2-2. 統計カード（4つ）
- [ ] 「全ユーザー数」カードに総数が表示されること
- [ ] 「今月新規」カードに今月登録数（`+N`形式）が表示されること
- [ ] 前月に登録があれば「▲〇%」or「▼〇%」の前月比が表示されること
- [ ] 前月に登録が0件なら前月比バッジは表示されないこと
- [ ] 「アクティブ(30日)」カードに30日以内の `current_sign_in_at` を持つユーザー数が表示されること
- [ ] 「停止中」カードに `suspended_at` を持つユーザー数が表示されること

### 2-3. フィルターバー
- [ ] 検索フィールドに名前の一部を入れて検索 → 該当ユーザーのみ表示されること
- [ ] 検索フィールドにメールアドレスの一部を入れて検索 → 該当ユーザーのみ表示されること
- [ ] 検索フィールドに「%」や「_」を入れても全件マッチしないこと（メタ文字エスケープの確認）
- [ ] ステータス「全て」を選択 → 全員が表示されること
- [ ] ステータス「正常」を選択 → 稼働中ユーザーのみが表示されること
- [ ] ステータス「停止」を選択 → 停止中ユーザーのみが表示されること
- [ ] 登録期間の from / to を指定して検索 → 指定期間に登録されたユーザーのみ表示されること
- [ ] 日付の from / to の片方だけでも機能すること
- [ ] 不正な日付を叩き込まれた場合（例: URL直接 `?from=abc`）、「日付の形式が正しくありません」のアラートが表示されること
- [ ] 「リセット」をクリック → フィルターが解除され全件表示に戻ること

### 2-4. テーブル表示
- [ ] ヘッダーに「ID / ユーザー / 登録日 / 最終ログイン / ステータス / アクション」が表示されること
- [ ] IDが5桁ゼロ埋め（例: `#00012`）で表示されること
- [ ] ユーザー列にイニシャルアバター + 名前 + メールが表示されること
- [ ] アバターの背景色がユーザーごとに異なること
- [ ] 絵文字を含む名前（例: 「👨太郎」）でもイニシャルが壊れないこと（`👨` が表示される）
- [ ] 登録日が `YYYY/MM/DD` 形式で表示されること
- [ ] 最終ログインが「〇時間前 / 昨日 / 〇日前」形式で表示されること
- [ ] 一度もログインしていないユーザーは「-」と表示されること
- [ ] 稼働中ユーザーは緑の「正常」バッジが表示されること
- [ ] 停止中ユーザーは赤の「停止」バッジが表示されること
- [ ] 各行の「詳細」リンクをクリック → 詳細ページに遷移すること

### 2-5. ページネーション
- [ ] 21件以上のデータがある場合、ページネーションが表示されること
- [ ] デフォルトで20件ずつ表示されること
- [ ] ページ番号をクリック → 該当ページに遷移すること
- [ ] 「次へ」「前へ」ボタンが機能すること
- [ ] 「全 N 件中 X〜Y 件表示」が正しい数値であること
- [ ] 検索/フィルタ中にページ遷移しても絞り込み条件が維持されること

### 2-6. 該当ユーザーなし
- [ ] 存在しないキーワードで検索 → 「該当するユーザーが見つかりませんでした」が表示されること

## 3. ユーザー詳細ページ（`/admin/users/:id`）

### 3-1. ページ表示
- [ ] パンくずリスト「ホーム > ユーザー管理 > ユーザー名」が表示されること
- [ ] 「ユーザー管理」リンクをクリック → 一覧に戻れること
- [ ] ページタイトルにユーザー名が表示されること
- [ ] 「ユーザーアカウントの詳細情報と学習履歴を管理します。」のサブコピーが表示されること

### 3-2. プロフィールカード
- [ ] 96pxのイニシャルアバター（`name` の先頭1文字）が表示されること
- [ ] 稼働中ユーザーはアバター右下に緑の稼働インジケーターが表示されること
- [ ] 停止中ユーザーはインジケーターが非表示になること
- [ ] 名前（大）+ ステータスバッジ（正常/停止）が表示されること
- [ ] メールアドレスが表示されること
- [ ] 登録日が `YYYY/MM/DD` 形式で表示されること
- [ ] 最終ログインが `YYYY/MM/DD` 形式で表示されること
- [ ] 未ログインユーザーは最終ログインが「-」と表示されること

### 3-3. タブナビゲーション
- [ ] 「クイズ結果」「お気に入り」の2タブが表示されること
- [ ] デフォルトで「クイズ結果」タブがアクティブ（青色下線）になっていること
- [ ] 「お気に入り」をクリック → URL が `?tab=favorites` に変わりタブコンテンツが切り替わること
- [ ] 「クイズ結果」をクリック → URLが `?tab=quiz_results` に変わりタブコンテンツが切り替わること
- [ ] 不正なtabパラメータ（例: `?tab=hack`）を指定 → クイズ結果タブが表示される（フォールバック）

### 3-4. クイズ結果タブ
- [ ] ヘッダーに「受験日 / クイズ / 状態 / スコア」が表示されること
- [ ] 実行済みのクイズ結果が新しい順に表示されること
- [ ] 受験日は `YYYY/MM/DD` 形式で表示されること
- [ ] 完了ステータスは「完了」バッジ（緑）で表示されること
- [ ] 進行中ステータスは「進行中」バッジ（グレー）で表示されること
- [ ] スコアが表示されること（無ければ「-」）
- [ ] クイズ結果が0件の場合「クイズ結果がありません」が表示されること
- [ ] 21件以上ある場合、ページネーションが表示されること
- [ ] ページ移動してもタブ状態（クイズ結果）が維持されること

### 3-5. お気に入りタブ
- [ ] ヘッダーに「追加日 / 対象 / 種別」が表示されること
- [ ] お気に入りが新しい順に表示されること
- [ ] 追加日は `YYYY/MM/DD` 形式で表示されること
- [ ] Character をお気に入りにしたものは、対象列に人物名(`name`)が表示されること
- [ ] Event をお気に入りにしたものは、対象列にイベント名(`title`)が表示されること
- [ ] 種別が「人物」（青バッジ）または「出来事」（アンバーバッジ）で表示されること
- [ ] お気に入りが0件の場合「お気に入りがありません」が表示されること
- [ ] 21件以上ある場合、ページネーションが表示されること
- [ ] ページ移動してもタブ状態（お気に入り）が維持されること

### 3-6. 存在しないユーザー
- [ ] `/admin/users/99999` など存在しないIDにアクセス → 404エラーになること

## 4. アカウント停止機能

### 4-1. 停止ボタンの表示
- [ ] 稼働中ユーザーの詳細ページで右上に赤枠の「アカウント停止」ボタンが表示されること
- [ ] **自分自身の詳細ページ**では「アカウント停止」ボタンが薄く（disabled）表示され、ツールチップ「自分自身は停止できません」が出ること
- [ ] 停止中ユーザーの詳細ページでは緑枠の「停止解除」ボタンが表示されること

### 4-2. 停止モーダル
- [ ] 「アカウント停止」ボタンをクリック → 確認モーダルが表示されること
- [ ] モーダルに「アカウントを停止しますか？」と対象ユーザー名が表示されること
- [ ] モーダル外（backdrop）をクリック、または`Esc`キーで閉じられること
- [ ] 「キャンセル」をクリック → モーダルが閉じ、停止されないこと
- [ ] 「停止する」をクリック → 詳細ページにリダイレクト、「〇〇を停止しました」の通知が出ること
- [ ] 停止後、プロフィールカードのステータスバッジが「停止」に変わること
- [ ] 停止後、右上ボタンが「停止解除」（緑枠）に変わること
- [ ] 停止後、稼働インジケーター（緑の丸）が非表示になること

### 4-3. 停止解除モーダル
- [ ] 停止中ユーザーの「停止解除」ボタンをクリック → 確認モーダルが表示されること
- [ ] モーダルに「停止を解除しますか？」と対象ユーザー名が表示されること
- [ ] 「キャンセル」をクリック → モーダルが閉じ、解除されないこと
- [ ] 「解除する」をクリック → 詳細ページにリダイレクト、「〇〇の停止を解除しました」の通知が出ること
- [ ] 解除後、ステータスバッジが「正常」に戻ること
- [ ] 解除後、右上ボタンが「アカウント停止」（赤枠）に戻ること

### 4-4. 自分自身の停止を試みる
- [ ] 管理者が自分自身のIDに対して `PATCH /admin/users/:id/suspend` を直接叩く → 停止されず「自分自身を停止することはできません」のアラートが出ること

## 5. 停止ユーザーのログイン制御

### 5-1. 停止後の新規ログイン
- [ ] 管理者が一般ユーザーを停止する
- [ ] ログアウトして、停止された一般ユーザーのメールとパスワードでログインを試みる
- [ ] ログイン失敗、ログインページに「このアカウントは停止されています。管理者にお問い合わせください。」のメッセージが表示されること

### 5-2. ログイン中ユーザーの即時切断
- [ ] 一般ユーザーとしてログイン中の状態で、別タブから管理者で該当ユーザーを停止する
- [ ] 一般ユーザーの元タブで任意のページ（例: `/profile`）に遷移 → ログイン画面にリダイレクトされること
- [ ] ログイン画面に「このアカウントは停止されています…」のアラートが表示されること

### 5-3. 解除後のログイン
- [ ] 停止解除されたユーザーが再度ログインを試みる → 正常にログインできること
