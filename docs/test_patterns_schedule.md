# スケジュール機能 手動テストパターン

## テスト準備
- テストアカウント: `test@example.com` / パスワードは設定済みのもの
- `./bin/dev` でサーバー起動済み

---

## 1. 未ログイン状態

### 1-1. ヘッダー確認
- [ ] ヘッダーに「スケジュール」リンクが表示されて**いない**こと

### 1-2. 直接URL アクセス
- [ ] `/schedules/new` にアクセス → ログインページにリダイレクトされること

---

## 2. ログイン〜ヘッダー

### 2-1. ログイン後のヘッダー確認
- [ ] ログイン後、ヘッダーに「スケジュール」リンクが表示されること
- [ ] 「スケジュール」をクリック → 新規作成フォームに遷移すること

---

## 3. スケジュール新規作成フォーム（`/schedules/new`）

### 3-1. フォーム表示確認
- [ ] ページタイトル「学習スケジュールの設定」が表示されること
- [ ] 説明文「自分のペースに合わせて〜」が表示されること
- [ ] 以下のセクションが表示されること：
  - [ ] 学習期間（開始日・終了日）
  - [ ] 1日の目標学習時間（ドロップダウン）
  - [ ] 週間スケジュール（曜日トグル 月〜日）
  - [ ] 学習ユニットの選択（ドロップダウン + タグ）
  - [ ] 学習メモ・目標（テキストエリア）
  - [ ] キャンセル / スケジュールを保存する ボタン

### 3-2. 学習期間の入力
- [ ] 開始日をカレンダーから選択できること
- [ ] 終了日をカレンダーから選択できること

### 3-3. 1日の目標学習時間
- [ ] ドロップダウンに1時間〜12時間の選択肢があること
- [ ] デフォルトが「1時間」になっていること

### 3-4. 週間スケジュール（曜日トグル）
- [ ] 初期状態ではどの曜日もOFF（未選択スタイル）であること
- [ ] 曜日をクリック → ONになり色が変わること（濃紺背景 + 白文字）
- [ ] もう一度クリック → OFFに戻ること
- [ ] 複数曜日を選択できること

### 3-5. 学習ユニットの選択
- [ ] ドロップダウンに「学習ユニットを選択してください...」と表示されていること
- [ ] ドロップダウンに全ユニット（ルネサンス、古代ギリシャ文化、バロック、ロマン主義）が表示されること
- [ ] ユニットを選択 → タグとして追加されること（水色背景）
- [ ] タグの「×」をクリック → タグが削除されること
- [ ] 選択済みのユニットはドロップダウンから非表示になること
- [ ] タグ削除後、ドロップダウンに再表示されること

### 3-6. 学習メモ・目標
- [ ] テキストエリアにプレースホルダーが表示されていること
- [ ] 自由にテキストを入力できること

---

## 4. スケジュール保存（正常系）

### 4-1. 全項目入力して保存
- [ ] 以下を入力して「スケジュールを保存する」をクリック：
  - 開始日: 2026/05/01
  - 終了日: 2026/07/31
  - 目標学習時間: 2時間
  - 曜日: 月・火・水・木・金
  - 学習ユニット: ルネサンス, 古代ギリシャ文化
  - メモ: 「夏休み前に基礎を固める」
- [ ] プロフィール画面にリダイレクトされること
- [ ] 「スケジュールを作成しました」のフラッシュメッセージが表示されること

### 4-2. 最小限の入力で保存
- [ ] 開始日・終了日・学習時間のみ入力（曜日・ユニット・メモは空）
- [ ] 保存できること

---

## 5. スケジュール保存（異常系）

### 5-1. 開始日が空
- [ ] 開始日を空にして保存 → エラーメッセージが表示されること

### 5-2. 終了日が空
- [ ] 終了日を空にして保存 → エラーメッセージが表示されること

### 5-3. 終了日が開始日より前
- [ ] 開始日: 2026/07/31、終了日: 2026/05/01 で保存
- [ ] エラーメッセージ「は開始日より後の日付にしてください」が表示されること

### 5-4. 終了日と開始日が同じ
- [ ] 同じ日付で保存 → エラーが表示されること

---

## 6. プロフィール画面でのスケジュール表示

### 6-1. スケジュール未作成の場合
- [ ] 「まだスケジュールが設定されていません」が表示されること
- [ ] 「スケジュールを作成する」ボタンが表示されること
- [ ] ボタンをクリック → 新規作成フォームに遷移すること

### 6-2. スケジュール作成済みの場合（1件）
- [ ] 「今後の学習スケジュール」セクションに以下が表示されること：
  - [ ] 学習期間（開始日 〜 終了日）
  - [ ] 1日の目標学習時間
  - [ ] 学習曜日（選択した曜日が濃紺、未選択がグレー）
  - [ ] 学習ユニット（タグ表示）
  - [ ] メモ内容
- [ ] 「編集する」リンクが表示されること
- [ ] 「削除する」ボタンが表示されること
- [ ] ページネーション表示が「1 / 1」になっていること

### 6-3. スケジュール作成済みの場合（複数件）
- [ ] ページネーション表示が「1 / N」（Nはスケジュール数）になっていること
- [ ] 「→」ボタンで次のスケジュールに切り替わること
- [ ] 「←」ボタンで前のスケジュールに切り替わること
- [ ] 最初のページで「←」ボタンが無効化されていること
- [ ] 最後のページで「→」ボタンが無効化されていること
- [ ] 開始日が直近のスケジュールが最初に表示されること
- [ ] 各スケジュールにそれぞれ「編集する」「削除する」が表示されること

### 6-4. スケジュール削除
- [ ] 「削除する」をクリック → 確認ダイアログが表示されること
- [ ] 確認ダイアログで「OK」 → スケジュールが削除されプロフィール画面に戻ること
- [ ] 「スケジュールを削除しました」のフラッシュメッセージが表示されること
- [ ] 確認ダイアログで「キャンセル」 → 削除されないこと
- [ ] 全スケジュール削除後 → 「まだスケジュールが設定されていません」が表示されること

---

## 7. スケジュール編集

### 7-1. 編集画面遷移
- [ ] プロフィール画面の「編集する」をクリック → 編集フォームに遷移すること
- [ ] ページタイトルが「学習スケジュールの編集」になっていること

### 7-2. 既存データの表示
- [ ] 保存済みの開始日・終了日が入力欄に反映されていること
- [ ] 保存済みの目標学習時間がドロップダウンで選択されていること
- [ ] 保存済みの曜日がON状態になっていること
- [ ] 保存済みの学習ユニットがタグとして表示されていること
- [ ] 保存済みのメモがテキストエリアに表示されていること

### 7-3. 内容を変更して更新
- [ ] 目標学習時間を3時間に変更
- [ ] 曜日を追加/削除
- [ ] 学習ユニットを追加/削除
- [ ] 「スケジュールを更新する」をクリック
- [ ] プロフィール画面にリダイレクトされること
- [ ] 「スケジュールを更新しました」のフラッシュメッセージが表示されること
- [ ] 変更内容が反映されていること

### 7-4. キャンセル
- [ ] 編集画面で「キャンセル」をクリック → プロフィール画面に戻ること
- [ ] 変更が保存されていないこと
