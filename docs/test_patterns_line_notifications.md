# LINE通知機能 手動テストパターン

## テスト準備
- LINE Developersコンソールで Messaging API用チャネル を作成済み
  - ログイン用とは別チャネル / 同じプロバイダ配下
  - 「Messaging API設定」→「チャネルアクセストークン(長期)」を発行
- `.env` に `LINE_MESSAGING_CHANNEL_ACCESS_TOKEN` を設定済み
- 動作検証用のLINEアカウントを2つ用意
  - A: Botを友達追加済み
  - B: Botを友達未追加 or ブロック済み
- 通常ユーザー(LINE未連携)とLINE連携ユーザーをDBに用意
- `docker compose up` でサーバー起動済み
- 開発環境のメール確認: `http://localhost:3000/letter_opener`

---

## 1. LINE連携ユーザーのスケジュール登録通知

### 1-1. 通知が届くこと
- [ ] LINE連携ユーザー(Botを友達追加済み)でログイン
- [ ] `/schedules/new` でスケジュールを新規登録
- [ ] 保存成功後、Bot からLINEメッセージが届くこと(数秒〜10秒程度)
- [ ] **letter_openerに該当ユーザー宛メールが届かないこと**(LINE優先で送らない仕様)

### 1-2. メッセージのヘッダ
- [ ] メッセージ先頭に「📚 学習予定を登録しました!」が表示
- [ ] 絵文字が文字化けせず正しく表示

### 1-3. メッセージの必須項目
- [ ] **■ 学習期間**: 開始日〜終了日 (例: `4月27日〜5月10日`)
- [ ] **■ 1日の目標学習時間**: `〇時間` (例: `2時間`)

### 1-4. オプション項目(入力時のみ表示)
- [ ] **学習曜日**を選択した場合 → 「■ 学習曜日」セクションに `月、水、金` の形式で表示
- [ ] **学習ユニット**を選択した場合 → 「■ 学習ユニット」セクションに `古代日本史、中世ヨーロッパ史` のようにカンマ区切りで表示
- [ ] **メモ**を入力した場合 → 「■ メモ・目標」セクションにメモ内容が表示

### 1-5. 未入力項目は非表示
- [ ] 曜日を選択しなかった場合 → 「■ 学習曜日」セクションが**表示されない**
- [ ] ユニットを選択しなかった場合 → 「■ 学習ユニット」セクションが**表示されない**
- [ ] メモを入力しなかった場合 → 「■ メモ・目標」セクションが**表示されない**

### 1-6. リンクの動作
- [ ] メッセージ末尾に「詳細はこちら:」と URL が表示
- [ ] URL が `https://culturelink.jp/profile` (本番) または `http://localhost:3000/profile` (開発)
- [ ] URLをタップ → ブラウザで `/profile` が開く(ログイン済み状態)

---

## 2. 通常ユーザー(LINE未連携)のスケジュール登録通知(回帰テスト)

### 2-1. メールが届くこと
- [ ] 通常ユーザー(provider未設定 or google_oauth2)でログイン
- [ ] `/schedules/new` でスケジュールを新規登録
- [ ] `http://localhost:3000/letter_opener` にメールが届く
- [ ] **LINEメッセージは届かないこと**(連携してないため当然送らない)

### 2-2. メール内容(既存仕様の回帰確認)
- [ ] 件名: `【culture-link】学習予定を登録しました`
- [ ] 本文に学習期間/曜日/ユニット/メモが正しく表示
- [ ] プロフィールページへのリンクが動作

---

## 3. Bot未友達ユーザーへの送信(エラー系)

### 3-1. Botを友達追加していないLINE連携ユーザー
- [ ] Botを友達追加していないLINE連携ユーザーでログイン
- [ ] スケジュールを新規登録
- [ ] 通知は届かない (LINE側で `403 You cannot send messages...` を返すため)
- [ ] **アプリは正常完了**(画面遷移エラーなし)
- [ ] `tail -f log/development.log` で `[LineNotifier] push failed: status=403` のログが出ること

### 3-2. Botをブロックしたユーザー
- [ ] Botを友達追加 → スケジュール登録で通知が来ることを確認
- [ ] Botを「ブロック」する
- [ ] 再度スケジュールを新規登録
- [ ] 通知は届かない
- [ ] アプリは正常完了
- [ ] ログに 403 エラーが残ること

---

## 4. ジョブの非同期挙動

### 4-1. perform_laterで送信される
- [ ] `docker compose exec web rails console`
- [ ] `ActiveJob::Base.queue_adapter` が `:async`(dev) または `:test`(test) であること
- [ ] スケジュール登録後、ActiveJob のログにジョブ実行が記録されていること

### 4-2. リトライ挙動(任意・難)
- [ ] 一時的にネットワークを切断 or トークンを無効値に変更してエラー誘発
- [ ] ログにリトライ試行が複数回記録されること(retry_on で最大5回)
- [ ] 5回失敗後はエラーログを残してジョブ終了すること

### 4-3. スケジュール削除済みのジョブ
- [ ] スケジュール作成 → 即座に削除 → ジョブ実行
- [ ] `Schedule.find_by` がnil → 何もせず終了
- [ ] LINE APIへのリクエストが発生しないこと

## 5. メッセージ内容のバリエーション

### 5-1. 短期間スケジュール(同月)
- [ ] 4/27〜4/30 のような短期間で登録
- [ ] 表示: `4月27日〜4月30日`

### 5-2. 月をまたぐスケジュール
- [ ] 4/27〜5/10 のような月跨ぎで登録
- [ ] 表示: `4月27日〜5月10日`

### 5-3. 全曜日選択
- [ ] 月〜日 全曜日を選択
- [ ] 表示: `月、火、水、木、金、土、日`

### 5-4. 大量のメモ
- [ ] 改行を含む長文メモ(数百文字)を入力
- [ ] LINEメッセージの最大長(5000文字)に収まることを確認
- [ ] 改行が正しく表示されること

### 5-5. 大量の学習ユニット
- [ ] 10個以上のユニットを選択
- [ ] カンマ区切りで全ユニット名が表示されること

## 6. 環境変数の確認

### 6-1. 環境変数未設定時の挙動
- [ ] `.env` から `LINE_MESSAGING_CHANNEL_ACCESS_TOKEN` を一時的に削除して再起動
- [ ] LINE連携ユーザーでスケジュール登録
- [ ] エラーログに `KeyError: key not found: "LINE_MESSAGING_CHANNEL_ACCESS_TOKEN"` が出る
- [ ] アプリ自体は落ちず、ジョブが失敗するだけ

### 6-2. 無効なトークン
- [ ] `.env` のトークンを意図的に無効な値に変更
- [ ] スケジュール登録 → ログに `[LineNotifier] push failed: status=401` が出る
- [ ] アプリは正常完了

---

## 7. 本番環境(Render)固有の確認

### 7-1. 環境変数の設定
- [ ] Renderダッシュボードに `LINE_MESSAGING_CHANNEL_ACCESS_TOKEN` を設定済み
- [ ] LINE Developersで発行した**長期アクセストークン**を使用
- [ ] (将来Webhook実装時) `LINE_MESSAGING_CHANNEL_SECRET` も設定

### 7-2. 本番でのフロー確認
- [ ] 本番URL `https://culturelink.jp` でLINE連携ユーザーがログイン
- [ ] スケジュール登録
- [ ] LINE通知が届くこと
- [ ] メッセージのリンクが `https://culturelink.jp/profile` であること

### 7-3. 通常ユーザー(回帰)
- [ ] 通常ユーザーで登録 → メールが届く
- [ ] メールのリンクが本番URLであること

## トラブルシュート

| 症状 | 想定原因 | 確認方法 |
| --- | --- | --- |
| LINE通知が届かない | Bot未友達 / ブロック中 | LINEアプリで友達一覧確認 |
| 401エラー | アクセストークン不正 | LINE Developersで再発行 |
| 403エラー | ユーザーがBotブロック中 / 友達でない | LINE側のステータス確認 |
| 5xxエラー | LINE側障害 | `https://developers.line.biz/ja/news/` で障害情報確認 |
| URLが`localhost`になる | `default_url_options` 未設定 | `config/environments/<env>.rb` 確認 |
| ジョブが滞留する | `:async`アダプタの限界 | プロセス再起動 or Sidekiq導入検討 |
