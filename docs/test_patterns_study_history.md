# 学習履歴機能 手動テストパターン

対象機能:
- プロフィール画面の **学習履歴カード**（上部の3枚カード・ページネーション付き）
- プロフィール画面の **最近チェックした項目**（記事閲覧履歴・最大4件）

## テスト準備
- テストアカウント: `test@example.com` など既存のユーザー
- `./bin/dev` でサーバー起動済み（本番環境なら該当URL）
- seedデータ（クイズ / Event / Character）が投入済み
- プロフィール画面URL: `/profile`

### テスト前の履歴リセット（任意）
ゼロ状態から確認したい場合、以下のコマンドで対象ユーザーの履歴をリセット:

```bash
docker compose exec web bin/rails runner 'u = User.find_by(email: "test@example.com"); u.article_views.destroy_all; u.quiz_results.destroy_all; puts "reset"'
```

※本番環境では対象ユーザーを間違えないよう慎重に実行すること。

## 1. 未ログイン状態

### 1-1. プロフィール画面への直接アクセス
- [ ] `/profile` にアクセス → ログインページにリダイレクトされること

---

## 2. 学習履歴カード — 空状態

### 2-1. 受験済みクイズが0件のとき
- [ ] ユーザーの `quiz_results` が0件の状態でプロフィール画面を開く
- [ ] 上部に「学習履歴」の見出しが表示されること
- [ ] 「まだ受験済みのクイズがありません」のメッセージが表示されること
- [ ] 「クイズを受ける」ボタン（紫グラデーション）が表示されること
- [ ] 「クイズを受ける」をクリック → 小テスト一覧画面（`/quizzes`）に遷移すること
- [ ] カードのページネーション矢印は**表示されない**こと

### 2-2. 受験中（in_progress）のクイズしかない場合
- [ ] クイズを途中まで受けて、status が `in_progress` のまま残す（異常終了など）
- [ ] プロフィール画面に遷移
- [ ] そのクイズは**学習履歴カードに表示されない**こと（completed のみ表示）
- [ ] 空状態メッセージが表示されること

---

## 3. 学習履歴カード — 受験済みクイズあり

### 3-1. 1件だけ受験済みの場合
- [ ] クイズを1つ最後まで受験して結果を記録
- [ ] プロフィール画面を開く
- [ ] カード1枚だけ表示されること（2枠目、3枠目は空き）
- [ ] カードに以下が表示されること:
  - [ ] クイズタイトル
  - [ ] 「X% 完了」の表示
  - [ ] 右側に円形プログレス（スコアに応じた色）
  - [ ] 円形プログレスの中央にパーセンテージ数値
- [ ] ページネーション矢印は**表示されない**こと（1ページのみ）

### 3-2. 2〜3件受験済みの場合
- [ ] 2〜3件のクイズを受験
- [ ] カードが2〜3枚横並びで表示されること
- [ ] ページネーション矢印は**表示されない**こと（1ページのみ）

### 3-3. 4件以上受験済みの場合（ページネーション）
- [ ] 4件以上のクイズを受験
- [ ] 1ページ目に3枚、2ページ目に残りが表示されること
- [ ] 右上に「1 / N」のインジケーターが表示されること
- [ ] 左矢印が初期状態で無効化（グレーアウト）されていること
- [ ] 右矢印をクリック → 2ページ目に遷移し、インジケーターが「2 / N」に更新されること
- [ ] 左矢印が有効化され、右矢印が最終ページなら無効化されること
- [ ] 左矢印クリックで前のページに戻れること

### 3-4. 並び順（最近受験順）
- [ ] 複数のクイズを異なるタイミングで受験
- [ ] 直近に受験したクイズほど**左（先頭）**に表示されること
- [ ] 再受験すると、そのクイズが先頭に移動すること

### 3-5. スコアに応じた色分け
| スコア帯 | 円形プログレスの色 | 数値表示色 |
|---|---|---|
| 90〜100% | 緑 | 緑 |
| 70〜89%  | 青 | 青 |
| 50〜69%  | オレンジ | オレンジ |
| 0〜49%   | 赤 | 赤 |

- [ ] 各スコア帯で異なる色が正しく表示されること（再受験で確認）

### 3-6. 長いタイトルの truncate
- [ ] 長いタイトルのクイズを用意（または一時的に既存クイズのタイトルを書き換え）
- [ ] カード内でタイトルが1行に収まり、末尾が `…` で省略されること
- [ ] 折り返しで高さが崩れないこと
- [ ] 円形プログレスが右端に正しく配置されていること

### 3-7. 他ユーザーのクイズ結果が混ざらない
- [ ] 別アカウントでログインし、別のクイズを受験
- [ ] 元アカウントに戻る → 元アカウントで受験したクイズのみ表示されること

---

## 4. 最近チェックした項目 — 空状態

### 4-1. 閲覧履歴が0件のとき
- [ ] ユーザーの `article_views` が0件の状態でプロフィール画面を開く
- [ ] 右カラムに「最近チェックした項目」の見出しが表示されること
- [ ] 「直近に確認した記事はありません」のメッセージが表示されること
- [ ] 書類アイコン（グレー）が上に表示されること

---

## 5. 最近チェックした項目 — 閲覧記録の動作

### 5-1. Event を閲覧して記録される
- [ ] 記事一覧画面（`/articles`）を開く
- [ ] 任意の Event 記事カードを**クリックして開く**
- [ ] プロフィール画面に戻る
- [ ] 「最近チェックした項目」にその Event が先頭に表示されること
- [ ] サムネイル（画像あれば画像、なければアイコン）が表示されること
- [ ] タイトル / 「出来事 • X年」 / 閲覧日時が表示されること
- [ ] カード全体が Event 詳細ページへのリンクになっていること

### 5-2. Character を閲覧して記録される
- [ ] 任意の Character 記事カードを**クリックして開く**
- [ ] プロフィール画面に戻る
- [ ] 「最近チェックした項目」にその Character が表示されること
- [ ] タイトル / 「人物 • X年」 / 閲覧日時が表示されること
- [ ] カード全体が Character 詳細ページへのリンクになっていること

### 5-3. Event と Character が混在表示される
- [ ] Event と Character を1つずつ閲覧
- [ ] 両方が「最近チェックした項目」に表示されること
- [ ] 最新閲覧順で並ぶこと

### 5-4. 同じ記事を複数回閲覧しても重複しない
- [ ] 同じ Event を2〜3回開く（プロフィール画面経由で往復）
- [ ] 「最近チェックした項目」に**重複せず1件だけ**表示されること
- [ ] リストの先頭に常に浮き上がってくること（順序だけ更新）

### 5-5. 最大4件制限
- [ ] 5件以上の異なる記事を順番に閲覧
- [ ] 「最近チェックした項目」に表示されるのは**最新4件だけ**であること
- [ ] 5件目に閲覧した古い記事はリストから消えること

### 5-6. 他ユーザーの閲覧履歴は混ざらない
- [ ] 別アカウントでログインして記事を閲覧
- [ ] 元アカウントに戻る → 元アカウントで閲覧した記事のみ表示されること

---

## 6. Turbo prefetch の誤記録防止（重要）

### 6-1. カードへのホバーでは記録されない
- [ ] 履歴をリセット（`article_views` を0件に）
- [ ] 記事一覧画面を開く
- [ ] 任意のカードに**マウスカーソルを数秒載せる**（クリックはしない）
- [ ] 複数のカードを順番にホバーする
- [ ] プロフィール画面に戻る
- [ ] 「最近チェックした項目」に**何も表示されない**こと
- [ ] 「直近に確認した記事はありません」が表示されていること

### 6-2. 実クリックで記録される
- [ ] 続けて、ホバーしていたカードの1つを**実際にクリックして開く**
- [ ] プロフィール画面に戻る
- [ ] 「最近チェックした項目」にクリックしたカードだけが表示されること
- [ ] 他のホバーしたカード（クリックしていないもの）は表示されないこと

### 6-3. タイムライン画面からの遷移
- [ ] タイムライン画面（`/timelines/:id`）を開く
- [ ] タイムライン上の Event リンクをクリックして詳細を開く
- [ ] プロフィール画面で「最近チェックした項目」に記録されていること

### 6-4. イベント詳細 → 関連人物リンクからの遷移
- [ ] Event 詳細ページを開く
- [ ] 関連する Character のリンクをクリック
- [ ] プロフィール画面で Event と Character の両方が記録されていること

---

## 7. ログイン/未ログインの切り分け

### 7-1. 未ログインで記事を閲覧 → 記録されない
- [ ] ログアウト
- [ ] 記事詳細ページ（`/events/:id` や `/characters/:id`）を直接開く
- [ ] ログインしてプロフィール画面を開く
- [ ] 未ログイン時の閲覧は**記録されていない**こと

### 7-2. ログイン状態で閲覧 → 記録される
- [ ] ログインした状態で記事を閲覧
- [ ] プロフィール画面で記録されていること

---

## 8. 他機能との併存確認

### 8-1. スケジュールカードのページネーション
- [ ] 学習履歴カードのページネーションと、スケジュールカードのページネーションが**独立して動く**こと
- [ ] 両方のカードにページネーション矢印が正しく表示されること
- [ ] 片方を操作してももう片方の表示が変化しないこと

### 8-2. プロフィール編集リンク
- [ ] 右上の「プロフィール編集」ボタンが引き続き機能すること

### 8-3. お気に入り機能との両立
- [ ] 記事詳細ページからお気に入り追加/解除が引き続き機能すること
- [ ] お気に入り追加しても `article_views` の記録とは独立していること

---

## 9. エッジケース

### 9-1. 削除された記事の閲覧履歴
- [ ] 閲覧済みの Event を DB から削除（`Event.find(x).destroy`）
- [ ] プロフィール画面を開く
- [ ] 該当の `article_view` レコードも `dependent: :destroy` で削除されていること
- [ ] エラーが起きず、他の履歴は正常に表示されること

### 9-2. プロフィール画面のリロード
- [ ] プロフィール画面で何度もリロードしても表示が安定していること
- [ ] ページネーションの状態は毎回1ページ目に戻ること（ステートレスでOK）
