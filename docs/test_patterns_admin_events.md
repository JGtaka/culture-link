# 管理画面 出来事管理 手動テストパターン

## テスト準備
- 管理者アカウント: `TestUser`（role=admin）でログイン
- `docker compose up` でサーバー起動済み

---

## 1. アクセス認可

### 1-1. 未ログイン
- [ ] `/admin/events` にアクセス → ログインページへリダイレクトされること

### 1-2. 一般ユーザー
- [ ] `/admin/events` にアクセス → ルートへリダイレクトされること
- [ ] 「管理者権限が必要です」のフラッシュメッセージが表示されること
- [ ] 直接 `DELETE /admin/events/:id` を叩いても削除されずリダイレクトされること

### 1-3. 管理者
- [ ] `/admin/events` にアクセス → 一覧ページが表示されること

---

## 2. 出来事一覧ページ（`/admin/events`）

### 2-1. ページ表示
- [ ] パンくずリスト「ホーム / コンテンツ / 出来事」が表示されること
- [ ] 「出来事一覧」タイトルとサブコピーが表示されること
- [ ] 「全 N 件」バッジの件数が実データと一致すること
- [ ] 「+ 新規追加」ボタンが表示されること

### 2-2. テーブル表示
- [ ] ヘッダーに「画像 / タイトル / 発生年 / 時代 / 地域 / 更新日 / 操作」が表示されること
- [ ] 出来事データがテーブルに一覧表示されること
- [ ] ActiveStorageの画像がサムネとして表示されること
- [ ] ActiveStorage画像がない場合、image_urlの画像がフォールバック表示されること
- [ ] 両方ない場合、プレースホルダーアイコンが表示されること
- [ ] 時代がバッジ形式で表示されること
- [ ] タイトルが長くても列幅が崩れないこと（truncateされる）

### 2-3. フィルター
- [ ] 「時代で絞り込み」ドロップダウンを選択 → 自動で絞り込みされること
- [ ] 「地域で絞り込み」ドロップダウンを選択 → 自動で絞り込みされること
- [ ] 「カテゴリで絞り込み」ドロップダウンを選択 → 自動で絞り込みされること
- [ ] 複数条件を組み合わせて絞り込みができること
- [ ] 「リセット」をクリック → フィルターが解除され全件表示に戻ること
- [ ] 絞り込んで行が減っても画像列の幅が縮まないこと（`table-fixed` + `colgroup`による固定）

### 2-4. ページネーション
- [ ] 20件を超えるデータがある場合、ページネーションが表示されること
- [ ] ページ番号をクリック → 該当ページに遷移すること
- [ ] 「次へ」ボタンが機能すること
- [ ] 「全 N 件中 X〜Y 件表示」が正しい数値であること

### 2-5. 操作ボタン
- [ ] 編集ボタン（ペンアイコン）をクリック → 編集ページに遷移すること
- [ ] 削除ボタン（ゴミ箱アイコン）をクリック → 確認ダイアログが表示されること
- [ ] 確認ダイアログで「OK」→ 出来事が削除され、一覧に戻ること
- [ ] 確認ダイアログで「キャンセル」→ 削除されないこと
- [ ] 削除後「〇〇を削除しました」のフラッシュメッセージが表示されること

---

## 3. 出来事作成ページ（`/admin/events/new`）

### 3-1. ページ表示
- [ ] パンくずリスト「ホーム / 出来事 / 新規作成」が表示されること
- [ ] 「出来事の登録」タイトルが表示されること
- [ ] 説明文「歴史的な出来事や文化的な節目をデータベースに登録します。」が表示されること
- [ ] 3カラムレイアウト（左2/3 + 右1/3）で表示されること
- [ ] 各セクションカードに青い左ボーダーがあること

### 3-2. 基本情報フォーム
- [ ] タイトル（テキスト入力）が表示されること
- [ ] 発生年（数値入力、プレースホルダー「例: 1008」）が表示されること
- [ ] 時代区分（ドロップダウン）に全時代が選択肢として表示されること
- [ ] 地域（ドロップダウン）に全地域が選択肢として表示されること
- [ ] カテゴリー（ドロップダウン）に全カテゴリーが選択肢として表示されること
- [ ] 学習単位（ドロップダウン）に全学習単位が選択肢として表示されること

### 3-3. 関連人物タグUI
- [ ] 入力欄に「紫式部」と入れると候補が出ること（部分一致）
- [ ] 入力欄が空のときは候補が出ないこと
- [ ] 候補をクリック → pill（青い×付きタグ）として追加されること
- [ ] pill の `×` をクリック → 削除されること
- [ ] 追加済みの人物は候補から消えること（重複防止）
- [ ] ⚠️ **Enter キーを押してもフォーム送信されないこと**
- [ ] ⚠️ **候補が開いた状態で外をクリック → 候補が閉じること**
- [ ] ⚠️ **候補が開いた状態で ESC キー → 候補が閉じること**
- [ ] 候補は最大10件まで表示されること

### 3-4. 本文フォーム
- [ ] 本文（テキストエリア）が表示されること
- [ ] 長文を入力でき、改行が保持されること
- [ ] 上下にリサイズできること（`resize-y`）

### 3-5. 画像アップロード
- [ ] 画像カードが表示されること
- [ ] 「クリックまたはドラッグ&ドロップ」エリアが表示されること
- [ ] 「JPG, PNG, WebP (最大 5MB)」の注釈が表示されること
- [ ] エリアをクリック → ファイル選択ダイアログが開くこと
- [ ] 画像を選択 → プレビューが表示されること
- [ ] 画像を選択 → ファイル名が表示されること
- [ ] 画像をドラッグ&ドロップ → プレビューが表示されること
- [ ] ドラッグ中にエリアが青くハイライトされること
- [ ] 「画像を変更する」ボタンで画像を再選択できること
- [ ] 画像クレジット入力欄が表示されること

### 3-6. 正常系：作成成功
- [ ] 全必須項目を入力し「作成する」をクリック
- [ ] 一覧ページにリダイレクトされること
- [ ] 「〇〇を作成しました」のフラッシュメッセージが表示されること
- [ ] 作成した出来事が一覧に表示されること
- [ ] アップロードした画像が一覧のサムネに表示されること
- [ ] 選択した関連人物が紐付いていること（詳細ページや編集ページで確認）

### 3-7. 異常系：バリデーションエラー
- [ ] タイトルを空で送信 → エラーメッセージが表示されること
- [ ] 発生年を空で送信 → エラーメッセージが表示されること
- [ ] 本文を空で送信 → エラーメッセージが表示されること
- [ ] 時代/地域/カテゴリー/学習単位のいずれか未選択で送信 → エラーが表示されること
- [ ] 5MBを超える画像を選択して送信 → エラーメッセージが表示されること
- [ ] 非画像ファイル（PDF等）を選択して送信 → エラーメッセージが表示されること
- [ ] エラー時、入力済みの値が保持されていること
- [ ] エラー時、選択した関連人物pillが保持されていること
- [ ] ⚠️ **画像を選択 + タイトル空で送信 → 500にならず422が返ること**（Active Storage未保存blobの回避確認）

### 3-8. キャンセル
- [ ] 「キャンセル」をクリック → 一覧ページに戻ること

### 3-9. 下部固定バー
- [ ] フォーム下部にスクロールしてもバーが表示され続けること
- [ ] フォームの最後のカードがバーの裏に隠れないこと

### 4-6. 画像の変更
- [ ] 「画像を変更する」ボタンで新しい画像を選択できること
- [ ] 新しい画像のプレビューが表示されること
- [ ] 更新後、新しい画像が一覧・詳細ページに反映されること

### 4-7. 異常系
- [ ] ⚠️ **画像を新規選択した状態でタイトルを空にして送信 → 500にならず422が返ること**（今回のバグ修正確認）
