# 管理画面 小テスト管理 手動テストパターン

## テスト準備
- 管理者アカウント: `TestUser`（role=admin）でログイン
- `docker compose up` でサーバー起動済み
- 事前に小テストカテゴリーを 2 つ以上用意しておくと確認しやすい（例: ルネサンス、バロック）

### 1-2. 一般ユーザー
- [ ] `/admin/quizzes` にアクセス → ルートへリダイレクトされること
- [ ] 「管理者権限が必要です」のフラッシュメッセージが表示されること
- [ ] 直接 `DELETE /admin/quizzes/:id` を叩いても削除されないこと
  - 確認方法: 一般ユーザーでログイン状態でブラウザの DevTools コンソールから以下を実行
    ```js
    fetch('/admin/quizzes/1', {
      method: 'DELETE',
      headers: {
        'X-CSRF-Token': document.querySelector('meta[name="csrf-token"]').content,
        'Accept': 'text/html'
      }
    }).then(r => console.log('status:', r.status, 'url:', r.url))
    ```
  - 期待: レスポンスURL が `/` になっていること、DBでQuizが残っていること
    ```bash
    docker compose exec web bin/rails runner "puts Quiz.find_by(id: 1)&.title || 'deleted'"
    ```
