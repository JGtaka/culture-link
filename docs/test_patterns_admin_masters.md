# 管理画面 マスター管理 手動テストパターン

対象ページ: `/admin/masters`（時代・地域・学習単元の一括管理）

## テスト準備
- 管理者アカウント: `TestUser`（role=admin）でログイン
- `docker compose up` でサーバー起動済み
- 事前にシード等で `Period` / `Region` / `StudyUnit` レコードを数件ずつ用意しておく
- 削除ガード確認用に、関連する `Event` / `Character` / `StudyUnitSchedule` も数件用意しておく

---

## 1. アクセス認可

### 1-1. 未ログイン
- [ ] `/admin/masters` にアクセス → ログインページへリダイレクトされること
- [ ] `POST /admin/periods` / `/admin/regions` / `/admin/study_units` を直接叩いても作成されずリダイレクトされること

### 1-2. 一般ユーザー
- [ ] `/admin/masters` にアクセス → ルートへリダイレクトされること
- [ ] 「管理者権限が必要です」のフラッシュメッセージが表示されること
- [ ] 直接 `DELETE /admin/periods/:id` を叩いても削除されずリダイレクトされること（地域・学習単元も同様）

### 1-3. 管理者
- [ ] `/admin/masters` にアクセス → マスター管理ページが表示されること

---

## 2. マスター管理ハブページ（`/admin/masters`）

### 2-1. ページ表示
- [ ] パンくずリスト「ホーム / マスター管理」が表示されること
- [ ] 「マスター管理」タイトルと「時代・地域・学習単元を一括で管理できます」サブコピーが表示されること
- [ ] 3つのカード（時代 / 地域 / 学習単元）が横並びで表示されること

### 2-2. カード共通の表示
- [ ] カードヘッダーに「マスター名」と「N件」の件数バッジが表示されること
- [ ] 件数バッジの数値が実レコード数と一致すること
- [ ] 一覧の各行に「名前」が表示されること
- [ ] 一覧の行にマウスホバー → 背景色が変わり、編集・削除アイコンが表示されること（通常時は非表示）
- [ ] カード下部に「+ 追加」ボタンが表示されること
- [ ] レコード0件のカードでは「まだ登録されていません」プレースホルダーが表示されること

### 2-3. 一覧のソート
- [ ] 各カード内の一覧が name 昇順で表示されていること（UTF-8順で許容）

---

## 3. サイドバー・ナビゲーション

- [ ] サイドバーに「マスター管理」リンクが追加されていること
- [ ] 「マスター管理」をクリック → `/admin/masters` に遷移すること
- [ ] マスター管理ページ表示時、「マスター管理」がアクティブ状態（青背景）になっていること
- [ ] 他の管理ページ（ダッシュボード・人物記事・出来事記事）では「マスター管理」は非アクティブであること

---

## 4. 新規追加（時代カードで代表確認、地域・学習単元も同様に確認）

### 4-1. インラインフォーム展開
- [ ] 時代カードの「+ 追加」をクリック → 同じ位置にフォームがインライン展開されること（ページ遷移しない）
- [ ] フォームに `name` 入力欄、「追加」ボタン、「キャンセル」リンクが表示されること
- [ ] ページを再読み込み → フォームは閉じていること（状態は保存されない）

### 4-2. 正常系：作成成功
- [ ] 名前を入力して「追加」→ `/admin/masters` にリダイレクトされること
- [ ] 「〇〇を作成しました」のフラッシュメッセージが表示されること
- [ ] 新しいレコードが時代カードの一覧に表示されること
- [ ] 件数バッジの数値が +1 されていること

### 4-3. 異常系：バリデーションエラー
- [ ] 名前を空で「追加」→ `/admin/periods/new` のフルページに遷移し、エラーメッセージが表示されること
- [ ] 既存と同じ名前で「追加」→ 同様にエラーメッセージ「Nameはすでに存在します」が表示されること
- [ ] エラー画面のパンくずに「ホーム / マスター管理 / 時代の追加」が表示されること
- [ ] 「キャンセル」リンクで `/admin/masters` に戻れること

### 4-4. 前後空白の自動除去
- [ ] 名前に先頭・末尾空白を入れて「追加」→ 保存されたレコードの名前から空白が除去されていること（例: `"  弥生時代  "` → `"弥生時代"`）
- [ ] 既存「縄文時代」があるとき「 縄文時代 」で追加 → 重複エラーになること

### 4-5. 地域カード
- [ ] 「+ 追加」→ 地域用のフォームが地域カード内に展開されること
- [ ] 作成後、地域カードのみに反映されること（時代/学習単元には入らない）

### 4-6. 学習単元カード
- [ ] 「+ 追加」→ 学習単元用のフォームが学習単元カード内に展開されること
- [ ] 作成後、学習単元カードのみに反映されること

---

## 5. 編集（時代カードで代表確認、地域・学習単元も同様に確認）

### 5-1. インラインフォーム展開
- [ ] 行をホバー → 編集アイコン（ペン）が表示されること
- [ ] 編集アイコンをクリック → その行がインラインフォームに切り替わること（他の行は表示のまま）
- [ ] フォームに現在の名前が初期値として入力されていること
- [ ] 「更新」ボタンと「キャンセル」リンクが表示されること

### 5-2. 正常系：更新成功
- [ ] 名前を変更して「更新」→ `/admin/masters` にリダイレクトされること
- [ ] 「〇〇を更新しました」のフラッシュメッセージが表示されること
- [ ] 該当行の名前が変わっていること

### 5-3. 異常系：バリデーションエラー
- [ ] 名前を空で「更新」→ `/admin/periods/:id/edit` のフルページに遷移し、エラーメッセージが表示されること
- [ ] 他の行と同じ名前に変更して「更新」→ 重複エラーが表示されること
- [ ] パンくずに「ホーム / マスター管理 / 時代の編集」が表示されること

### 5-4. キャンセル
- [ ] インラインフォーム状態で「キャンセル」→ `/admin/masters` に戻ること（フォームが閉じる）

### 5-5. 前後空白の自動除去
- [ ] 名前の先頭・末尾に空白を入れて「更新」→ 保存後の値から空白が除去されていること

---

## 6. 削除（時代カードで代表確認、地域・学習単元も同様に確認）

### 6-1. 確認ダイアログ
- [ ] ゴミ箱アイコンをクリック → 「〇〇を削除しますか？」の確認ダイアログが表示されること
- [ ] 「キャンセル」→ 削除されないこと
- [ ] 「OK」→ 削除が実行されること

### 6-2. 正常系：削除成功（関連レコード無し）
- [ ] 関連レコードが無い時代を削除 → `/admin/masters` にリダイレクトされること
- [ ] 「〇〇を削除しました」のフラッシュメッセージが表示されること
- [ ] 該当行が時代カードから消えていること
- [ ] 件数バッジの数値が -1 されていること

### 6-3. 異常系：削除ガード（Period）
- [ ] 関連 `Event` を持つ時代を削除しようとする → 削除されず、`/admin/masters` にリダイレクトされること
- [ ] 「Events に依存するレコードがあるため削除できません」等のアラートが表示されること
- [ ] 該当行が時代カードに残っていること
- [ ] 関連 `Character` を持つ時代でも同様に削除できないこと

### 6-4. 異常系：削除ガード（Region）
- [ ] 関連 `Event` を持つ地域を削除しようとする → 削除されずアラートが出ること
- [ ] 関連 `Character` を持つ地域でも同様に削除できないこと

### 6-5. 異常系：削除ガード（StudyUnit）
- [ ] 関連 `Event` を持つ学習単元を削除しようとする → 削除されずアラートが出ること
- [ ] 関連 `Character` を持つ学習単元でも同様に削除できないこと
- [ ] ⚠️ **ユーザーの `StudyUnitSchedule` に紐付く学習単元を削除しようとする → 削除されずアラートが出ること**（ユーザー学習計画の無通知削除を防ぐ仕様）

## 8. エッジケース・追加確認

- [ ] 3カードで同時に「+ 追加」を開いた場合、それぞれ独立して動作すること
- [ ] ある行を編集モードにした状態で、別の行の編集アイコンをクリック → 別の行もフォーム化されること（両方開いても干渉しない）
- [ ] 編集モード中に同じ行の削除ボタンをクリック → 削除できること
- [ ] 画面幅を狭くした時のレイアウト崩れがないこと（最低タブレット幅想定）
- [ ] ブラウザバック・フォワードでパンくずに違和感が出ないこと
