# 管理画面 人物管理 手動テストパターン

## テスト準備
- 管理者アカウント: `TestUser`（role=admin）でログイン
- `docker compose up` でサーバー起動済み

---

## 1. 人物一覧ページ（`/admin/characters`）

### 1-1. ページ表示
- [ ] パンくずリスト「ホーム / 人物」が表示されること
- [ ] 「人物一覧」タイトルが表示されること
- [ ] 「+ 新規追加」ボタンが表示されること

### 1-2. テーブル表示
- [ ] ヘッダーに「画像 / 名前 / 時代 / 関連出来事 / 更新日 / 操作」が表示されること
- [ ] 人物データがテーブルに一覧表示されること
- [ ] ActiveStorageの画像がアイコンとして表示されること
- [ ] ActiveStorage画像がない場合、image_urlの画像がフォールバック表示されること
- [ ] 両方ない場合、プレースホルダーアイコンが表示されること
- [ ] 時代がバッジ形式で表示されること
- [ ] 関連出来事の件数が正しく表示されること

### 1-3. フィルター
- [ ] 「時代で絞り込み」ドロップダウンを選択 → 自動で絞り込みされること
- [ ] 「学習単位で絞り込み」ドロップダウンを選択 → 自動で絞り込みされること
- [ ] 両方同時に絞り込みができること
- [ ] 「リセット」をクリック → フィルターが解除され全件表示に戻ること

### 1-4. ページネーション
- [ ] 20件を超えるデータがある場合、ページネーションが表示されること
- [ ] ページ番号をクリック → 該当ページに遷移すること
- [ ] 「次へ」ボタンが機能すること
- [ ] 「全 N 件中 X〜Y 件表示」が正しい数値であること

### 1-5. 操作ボタン
- [ ] 編集ボタン（ペンアイコン）をクリック → 編集ページに遷移すること
- [ ] 削除ボタン（ゴミ箱アイコン）をクリック → 確認ダイアログが表示されること
- [ ] 確認ダイアログで「OK」→ 人物が削除され、一覧に戻ること
- [ ] 確認ダイアログで「キャンセル」→ 削除されないこと
- [ ] 削除後「〇〇を削除しました」のフラッシュメッセージが表示されること

---

## 2. 人物作成ページ（`/admin/characters/new`）

### 2-1. ページ表示
- [ ] パンくずリスト「ホーム / 人物 / 新規作成」が表示されること
- [ ] 「人物を作成」タイトルが表示されること
- [ ] 説明文「新しく人物アーカイブを追加します〜」が表示されること

### 2-2. 基本情報フォーム
- [ ] 名前（テキスト入力）が表示されること
- [ ] 年代（テキスト入力、プレースホルダー「例: 1534~1582」）が表示されること
- [ ] 時代（ドロップダウン）に全時代が選択肢として表示されること
- [ ] 学習単位（ドロップダウン）に全学習単位が選択肢として表示されること
- [ ] 地域（ドロップダウン）に全地域が選択肢として表示されること

### 2-3. 説明・功績フォーム
- [ ] 説明（テキストエリア）が表示されること
- [ ] 功績（テキストエリア）が表示されること

### 2-4. 画像アップロード
- [ ] 画像カードが表示されること
- [ ] 「画像をドラッグ&ドロップ または クリックして選択」エリアが表示されること
- [ ] エリアをクリック → ファイル選択ダイアログが開くこと
- [ ] 画像を選択 → プレビューが表示されること
- [ ] 画像を選択 → ファイル名が表示されること
- [ ] 画像をドラッグ&ドロップ → プレビューが表示されること
- [ ] ドラッグ中にエリアが青くハイライトされること
- [ ] 「画像を変更する」ボタンで画像を再選択できること
- [ ] 画像クレジット入力欄が表示されること

### 2-5. 正常系：作成成功
- [ ] 全必須項目（名前、説明、功績、時代、学習単位、地域）を入力し「作成する」をクリック
- [ ] 一覧ページにリダイレクトされること
- [ ] 「〇〇を作成しました」のフラッシュメッセージが表示されること
- [ ] 作成した人物が一覧に表示されること
- [ ] アップロードした画像が一覧のアイコンに表示されること

### 2-6. 異常系：バリデーションエラー
- [ ] 名前を空で送信 → エラーメッセージが表示されること
- [ ] 説明を空で送信 → エラーメッセージが表示されること
- [ ] 功績を空で送信 → エラーメッセージが表示されること
- [ ] 5MBを超える画像を選択して送信 → エラーメッセージが表示されること
- [ ] エラー時、入力済みの値が保持されていること

### 2-7. キャンセル
- [ ] 「キャンセル」をクリック → 一覧ページに戻ること

### 2-8. 下部固定バー
- [ ] フォーム下部にスクロールしてもバーが表示され続けること
- [ ] フォームの最後のカードがバーの裏に隠れないこと

## 3. 人物編集ページ（`/admin/characters/:id/edit`）

### 3-1. ページ表示
- [ ] パンくずリスト「ホーム / 人物 / 〇〇 を編集」が表示されること
- [ ] 「〇〇 を編集」タイトルが表示されること
- [ ] 既存のデータがフォームに表示されていること

### 3-2. 既存画像の表示
- [ ] ActiveStorageの画像がある場合、プレビューに表示されること
- [ ] image_urlの画像がある場合、プレビューにフォールバック表示されること

### 3-3. 正常系：更新成功
- [ ] データを変更して「更新する」をクリック
- [ ] 一覧ページにリダイレクトされること
- [ ] 「〇〇を更新しました」のフラッシュメッセージが表示されること
- [ ] 変更内容が一覧に反映されていること

### 3-4. 画像の変更
- [ ] 「画像を変更する」ボタンで新しい画像を選択できること
- [ ] 新しい画像のプレビューが表示されること
- [ ] 更新後、新しい画像が一覧・詳細ページに反映されること
