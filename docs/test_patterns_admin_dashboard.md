# 管理画面ダッシュボード 手動テストパターン

## テスト準備
- 管理者アカウント: `TestUser`（role=admin）でログイン
- 一般ユーザーアカウント: 別のアカウントを用意（role=general）
- `docker compose up` でサーバー起動済み

---

## 1. アクセス制御

### 1-1. 未ログインでのアクセス
- [ ] `/admin` にアクセス → ログイン画面にリダイレクトされること

### 1-2. 一般ユーザーでのアクセス
- [ ] 一般ユーザー（role=general）でログイン後、`/admin` にアクセス
- [ ] トップページにリダイレクトされること
- [ ] 「管理者権限が必要です」のアラートが表示されること

### 1-3. 管理者ユーザーでのアクセス
- [ ] 管理者ユーザー（role=admin）でログイン後、`/admin` にアクセス
- [ ] ダッシュボード画面が正常に表示されること

---

## 2. ヘッダーの管理画面リンク（ユーザー向けサイト側）

### 2-1. 管理者ログイン時
- [ ] ヘッダーに青い「管理画面」ボタンが表示されること
- [ ] 「管理画面」ボタンをクリック → `/admin` に遷移しダッシュボードが表示されること

### 2-2. 一般ユーザーログイン時
- [ ] ヘッダーに「管理画面」ボタンが表示されて**いない**こと

### 2-3. 未ログイン時
- [ ] ヘッダーに「管理画面」ボタンが表示されて**いない**こと

---

## 3. ダッシュボード表示内容

### 3-1. ヘッダーセクション
- [ ] 「ダッシュボード」のタイトルが表示されること
- [ ] 「現在のシステムステータスとユーザーのアクティビティ概要」の説明文が表示されること

### 3-2. 統計カード（4つ）
- [ ] 「登録ユーザー数」カードが表示され、正しいユーザー数が表示されること
- [ ] 「今月の新規登録」カードが表示され、今月の登録数が表示されること
- [ ] 「公開記事数」カードが表示され、出来事＋人物の合計数が表示されること
- [ ] 「クイズ総数」カードが表示され、正しいクイズ数が表示されること

### 3-3. 人気の学習単位 TOP10
- [ ] 「人気の学習単位 TOP10」セクションが表示されること
- [ ] ヘッダー行に「RANK」「学習単位名」「ビュー数」が表示されること
- [ ] 最大5件の学習単位がランキング形式で表示されること
- [ ] 1位のランク番号が青色（#0058be）で表示されること
- [ ] ビュー数が3桁区切りで表示されること

### 3-4. 最近の登録ユーザー
- [ ] 「最近の登録ユーザー」セクションが表示されること
- [ ] 「表示」リンクが表示されること
- [ ] 最大4件のユーザーが表示されること
- [ ] 各ユーザーの名前・メールアドレス・登録日が表示されること
- [ ] 登録日が「YYYY/MM/DD」形式で表示されること

---

## 4. サイドバー

### 4-1. ロゴ
- [ ] 「Culture Link」のロゴとアイコンが表示されること
- [ ] 「Administrative Portal」のサブテキストが表示されること
- [ ] ロゴをクリック → ダッシュボード（`/admin`）に遷移すること

### 4-2. ナビゲーション
- [ ] 「ダッシュボード」「コンテンツ管理」「クイズ管理」「ユーザー管理」の4項目が表示されること
- [ ] ダッシュボードページでは「ダッシュボード」が青背景でアクティブ表示されること

### 4-3. フッターリンク
- [ ] 「サイトに戻る」リンクが表示されること
- [ ] 「サイトに戻る」をクリック → ユーザー向けトップページに遷移すること
- [ ] 「ログアウト」ボタンが表示されること
- [ ] 「ログアウト」をクリック → ログアウトしてトップページにリダイレクトされること

---

- [ ] 管理画面ヘッダー右側にログイン中の管理者名が表示されること
