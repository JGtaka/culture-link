# メール通知機能 手動テストパターン

## テスト準備
- テストアカウント（通常ユーザー）: `test@example.com`
- テストアカウント（Google連携ユーザー）: 任意のGoogleアカウント
- `docker compose up` でサーバー起動済み
- 開発環境は `letter_opener_web` を使用。メール確認URL: `http://localhost:3000/letter_opener`

---

## 1. 学習予定登録時のメール送信（通常ユーザー）

### 1-1. 基本の送信確認
- [ ] 通常ユーザーでログイン
- [ ] `/schedules/new` で新規スケジュールを作成
- [ ] 保存成功後、`http://localhost:3000/letter_opener` を開く
- [ ] 新しいメールが1件届いていること

### 1-2. メールヘッダーの確認
- [ ] **From**: `culture-link <noreply@mail.culturelink.jp>` と表示されること
- [ ] **To**: ログイン中ユーザーのメールアドレスになっていること
- [ ] **件名**: 「【culture-link】学習予定を登録しました」

### 1-3. メール本文の必須項目
- [ ] 「〇〇さん」の宛名（ユーザー名）が表示されていること
- [ ] 「学習期間」に、保存した開始日〜終了日が日本語で表示されていること
- [ ] 「1日の目標学習時間」に保存した時間（例:「2時間」）が表示されていること
- [ ] 「culture-linkをご利用いただきありがとうございます」の定型文があること

### 1-4. オプション項目（入力時のみ表示）
- [ ] **学習曜日**を選択した場合 → 「学習曜日」セクションに選択曜日が「月、火、…」形式で表示されること
- [ ] **学習ユニット**を選択した場合 → 「学習ユニット」セクションに選択ユニットが表示されること
- [ ] **メモ**を入力した場合 → 「メモ・目標」セクションにメモ内容が表示されること

### 1-5. 未入力項目は非表示
- [ ] 曜日を選択しなかった場合 → 「学習曜日」セクションが表示**されない**こと
- [ ] ユニットを選択しなかった場合 → 「学習ユニット」セクションが表示**されない**こと
- [ ] メモを入力しなかった場合 → 「メモ・目標」セクションが表示**されない**こと

### 1-6. リンク動作
- [ ] 「プロフィールページを開く」リンクが表示されていること
- [ ] リンクをクリック → ブラウザで `/profile` が開くこと（ログイン済み状態）

### 1-7. HTML版・テキスト版の表示確認
- [ ] `letter_opener` の画面上部で「HTML」「Plain」両方のタブが切り替えられること
- [ ] HTMLタブ: 整った表形式で項目が並ぶこと
- [ ] Plainタブ: 「■ 学習期間」等の記号付きでプレーンテキスト表示されること
- [ ] 両方とも日本語が文字化けせず表示されていること

---

## 2. 学習予定登録時のメール送信（Google連携ユーザー）

### 2-1. 通常ユーザーと同じメールが届くこと
- [ ] Google連携ユーザーでログイン
- [ ] スケジュールを作成
- [ ] letter_opener に、Googleアカウントのメアド宛てにメールが届いていること
- [ ] 本文の宛名がGoogleアカウントの氏名になっていること

---

## 3. メールが送信されない操作（編集・削除・失敗時）

### 3-1. スケジュール編集
- [ ] 既存スケジュールを編集・保存
- [ ] letter_opener に新規メールが**届かないこと**（編集時は通知なし）

### 3-2. スケジュール削除
- [ ] 既存スケジュールを削除
- [ ] letter_opener に新規メールが**届かないこと**

### 3-3. バリデーション失敗時
- [ ] 開始日を空のまま保存 → エラー画面に戻る
- [ ] letter_opener に新規メールが**届かないこと**

---

## 4. パスワード再発行メール（Devise標準機能 / 日本語化対応）

### 4-1. パスワード再発行リクエスト
- [ ] ログアウトした状態で `/users/sign_in` を開く
- [ ] 「パスワードを忘れましたか？」リンクをクリック
- [ ] `/users/password/new` でメールアドレスを入力して送信
- [ ] letter_opener にメールが届いていること

### 4-2. メール内容の確認
- [ ] **From**: `culture-link <noreply@mail.culturelink.jp>`
- [ ] 件名に「パスワード」関連の表記があること
- [ ] 本文が日本語で書かれていること
- [ ] 「〇〇様」の宛名（メアド）が表示されていること
- [ ] 「パスワードを再設定する」リンクが表示されていること
- [ ] 「有効期限は発行から◯時間です」の記載があること
- [ ] 「心当たりのない場合は、このメールを破棄してください」の注意書きがあること

### 4-3. リンク動作
- [ ] 「パスワードを再設定する」をクリック → パスワード変更フォームに遷移すること
- [ ] 新パスワードを入力して保存 → ログインページまたはトップページに遷移すること
- [ ] 新パスワードで実際にログインできること

---

## 5. メール送信の非同期動作

### 5-1. レスポンスの体感速度
- [ ] スケジュール作成時、保存直後すぐにプロフィール画面に遷移すること（メール送信の待ち時間を感じない）

### 5-2. メール送信失敗時のユーザー体験
（※開発環境では `letter_opener_web` を使うため通常失敗しないが、挙動の確認として）
- [ ] メール送信が万が一失敗しても、スケジュール自体は保存されていること
- [ ] エラー画面が出ず、通常のフロー通り動くこと

---

## 6. 退会後の関連メール処理（エッジケース）

### 6-1. スケジュール作成直後に退会した場合
- [ ] スケジュールを作成 → すぐに退会する
- [ ] エラーが発生せず、退会処理が完了すること
- [ ] letter_opener に届いたメール（もしあれば）を確認、例外で500にならないこと
